Introduction

Semantic analysis is the activity of a compiler to determine what
the types of various values are, how those types
interact in expressions, and whether those interactions are semantically reasonable. The purpose of semantic analysis is to check that we have a meaningful sequence of tokens.
Note that a sequence can be meaningful without being correct.in most programming languages,
the phrase "x + 1" would be considered to be a meaningful arithmetic expression.
However, if the programmer really meant to write "x - 1",
then it is not correct

Abstract

The core intent of this project is to understand in detail the process of Semantic Analysis in Compiler Design.
Semantic Analysis is one phase of design during which necessary semantic information from the source code is gathered.
It includes type checking, ensuring that variables are declared before use, ensuring number and type of parameters
match, and assures that all such rules of the language are satisfied. In our project we will be focusing on Semantic Analysis for C++.
Although we will not be implementing a code that checks of all rules of C++, we will be emulating two to three specific cases.
